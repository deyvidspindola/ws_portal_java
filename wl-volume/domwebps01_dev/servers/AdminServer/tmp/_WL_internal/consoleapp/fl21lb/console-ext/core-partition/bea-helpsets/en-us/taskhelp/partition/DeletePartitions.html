Delete domain partitions


When you delete a domain partition, all of the resource groups in the partition are deleted and all of the applications deployed to the partition are undeployed.

Note: If you accepted the default for the Preserve control when you created the domain partition, the partition file system is preserved when the partition is deleted.

To delete a domain partition:

  1. If you have not already done so, in the Change Center of the Administration Console, click Lock & Edit (see Use the Change Center).
  2. In the left pane of the Console, select Domain Partitions.

    The Domain Partitions table displays all of the partitions defined in the domain.
  3. Select the Control page.
  4. Select the partition and click Shutdown. You cannot delete a running partition.
  5. Select the Configuration page.
  6. Select the check box next to the domain partition you want to delete.
  7. Click Delete.
  8. Click Yes to confirm that you want to delete the domain partition.
  9. To activate these changes, in the Change Center of the Administration Console, click Activate Changes.
    Not all changes take effect immediately—some require a restart (see Use the Change Center).
