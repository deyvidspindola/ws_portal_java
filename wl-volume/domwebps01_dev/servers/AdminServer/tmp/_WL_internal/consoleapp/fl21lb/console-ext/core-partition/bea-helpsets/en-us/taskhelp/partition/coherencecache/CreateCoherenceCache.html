Create a Coherence cache at the partition level


Multi-tenant domains use a single, shared Coherence cluster that is available across all domain partitions. A single Coherence GAR is deployed across multiple domain partitions and used by all tenants. Isolation is provided at the domain partition level and is transparent to the application.

To define a Coherence cache to be used by this partition:

  1. If you have not already done so, in the Change Center of the Administration Console, click Lock & Edit (see Use the Change Center).
  2. In the left pane of the Console, select Domain Partitions.
  3. Select the partition for which you want to create the Coherence cache.
  4. Select the Coherence Caches page.
  5. Click New. The Create a New Coherence Cache page displays.
  6. In the Name field, enter the name that will identify this Coherence cache entry.
  7. In the Coherence Cache Name field, enter the name of the Coherence cache.
  8. In the GAR Application Name field, enter the name of the GAR that contains the cache configuration file that defines the scheme for this cache.
  9. Select the Shared option, if the cache is used by multiple domain partitions. Each partition that uses the cache must also define a Coherence cache configuration with the same configuration values and enable the Shared option.
  10. On the Create a New Coherence Cache page, enter the name that will identify your Coherence cache.
  11. In the Notes field, enter additional information about this Coherence cache entry as required.
  12. Click OK.
  13. To activate these changes, in the Change Center of the Administration Console, click Activate Changes.
    Not all changes take effect immediately—some require a restart (see Use the Change Center).

After you finish

To override cache configuration properties specific for this partition, See Override Coherence cache configuration properties at the partition level.

To enable cache sharing for this partition, See Enable Coherence cache sharing at the partition level.
