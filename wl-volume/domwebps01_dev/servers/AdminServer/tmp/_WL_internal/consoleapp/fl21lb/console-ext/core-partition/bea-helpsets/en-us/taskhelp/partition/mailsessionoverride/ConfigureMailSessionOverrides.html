Configure mail session overrides

Before you begin

Create mail session overrides


A mail session override defines partition-specific mail session attribute overrides of a mail session referenced by a partition resource group definition.

To configure a mail session override:

  1. In the left pane of the Console, select Domain Partitions.

    The Domain Partitions table displays all the partitions defined in the domain.

  2. Click on the name of the domain partition for which you want to configure a mail session override.
  3. Select the Resource Overrides > Mail Sessions configuration page.

    The Mail Session Overrides table displays all the mail session overrides that have been configured in this domain partition.

  4. Click on the name of the mail session override you want to configure.
  5. On the Settings > Configuration page, optionally provide partition-specific values in the Session Username, Session Password, and JavaMail Properties fields and click Save.

    The mail session override configuration is changed.

After you finish

If the resource is running, you must restart the partition for the overrides to take effect.
