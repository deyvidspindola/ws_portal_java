Create mail session overrides

Before you begin

Prior to creating a mail session override, you must have previously created a mail session. See Configure access to JavaMail.


Resource overriding allows administrators to customize resources at the partition level. A mail session override defines partition-specific mail session attribute overrides of a mail session referenced by a partition resource group definition.

To create a mail session override:

  1. In the left pane of the Console, select Domain Partitions.

    The Domain Partitions table displays all the partitions defined in the domain.

  2. Click on the name of the domain partition for which you want to create a mail session override.
  3. Select the Resource Overrides > Mail Sessions configuration page.

    The Mail Session Overrides table displays all the mail session overrides that have been configured in this domain partition.

  4. Click New.
  5. On the Create a Mail Session Override page, provide a Name for the mail session override.

    The system matches overrides using the name of the overriding configuration MBean with the name of the resource MBean.

  6. Optionally, in the Session Username, Session Password, and JavaMail Properties fields, provide partition-specific values and click OK.

    The new mail session override appears in the Mail Session Overrides table.

After you finish

If the resource is running, you must restart the partition for the overrides to take effect.
