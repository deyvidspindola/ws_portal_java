Associate partitions with Partition Work Managers

Before you begin

Define Partition Work Managers

Configure Partition Work Managers


A domain partition can be associated with (referenced by) only one Partition Work Manager. You can choose to either reference an exisitng one or to create one that is not shared by any other domain partition.

To associate a domain partition with a Partition Work Manager:

  1. If you have not already done so, in the Change Center of the Administration Console, click Lock & Edit (see Use the Change Center).
  2. In the left pane of the Console, select Domain Partitions.

    The Domain Partitions table displays all the partitions defined in the domain.

  3. Click on the name of the domain partition that you want to associate with a Partition Work Manager.
  4. Select the Partition Work Manager configuration page.
  5. From the Partition Work Manager menu, select an existing Partition Work Manager to use in your partition.

    Partition Work Managers can be used in more than one domain partition. However, a domain partition can be associated with only one Partition Work Manager.

  6. Select None to configure Partition Work Manager attributes for use in this partition only.

    See Configure Partition Work Managers

  7. Click Save.
  8. To activate these changes, in the Change Center of the Administration Console, click Activate Changes.
    Not all changes take effect immediately—some require a restart (see Use the Change Center).

Result

Partition Work Managers defined at the domain level and referenced by a partition are shown in the Partition Work Managers table. Partition Work Managers created within the partition configuration are not displayed in the Partition Work Managers table. However, both are persisted in the config.xml file.
