Override Coherence cache configuration properties at the partition level


Cache configuration properties can be specified for each domain partition. Partition-specific values are injected into the cache configuration when a GAR module is initialized, which allows the same GAR and cache configuration file to be deployed across partitions.

You cannot override Coherence cache configuration properties if a cache is shared across multiple partitions.

To override cache configuration properties with partition-specific values:

  1. If you have not already done so, in the Change Center of the Administration Console, click Lock & Edit (see Use the Change Center).
  2. In the left pane of the Console, select Domain Partitions.
  3. Select the partition where the Coherence cache is defined.
  4. Select the Coherence Caches page.
  5. From the Coherence Caches table, select a Coherence cache entry to modify its setting. The Settings for stores-shared-cache page displays.
  6. From the Properties set for this cache table, click New. You can also edit and delete existing property overrides. The Create a New Coherence Cache Property page displays.
  7. In the Property Name field, enter the name of a Coherence cache configuration property.
  8. In the Property Value field, enter a value that will be applied to the cache in this partition.
  9. In the Notes field, enter additional information about the property settings as required.
  10. Click OK.
  11. To activate these changes, in the Change Center of the Administration Console, click Activate Changes.
    Not all changes take effect immediately—some require a restart (see Use the Change Center).
