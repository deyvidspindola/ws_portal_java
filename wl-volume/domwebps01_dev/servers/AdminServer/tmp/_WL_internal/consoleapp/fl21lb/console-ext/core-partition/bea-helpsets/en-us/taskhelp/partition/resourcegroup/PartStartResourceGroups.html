Start resource groups at the partition level


Starting a resource group at the partition level starts any applications deployed to the resource group that can be started.

To start a resource group at the partition level:

  1. If you have not already done so, in the Change Center of the Administration Console, click Lock & Edit (see Use the Change Center).
  2. In the left pane of the Console, select Domain Partitions.
  3. On the Summary of Domain Partitions page, select the partition for which you want to start the resource group.
  4. Select the Resource Groups page.
  5. Select the Control page.
  6. Select the resource group you want to start.

    The resource group must have at least one target for it to be started. See Target resource groups at the partition level to virtual targets

  7. Click Start.
  8. To activate these changes, in the Change Center of the Administration Console, click Activate Changes.
    Not all changes take effect immediately—some require a restart (see Use the Change Center).
