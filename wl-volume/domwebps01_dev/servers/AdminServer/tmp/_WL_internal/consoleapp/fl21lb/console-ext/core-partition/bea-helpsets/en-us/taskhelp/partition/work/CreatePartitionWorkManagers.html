Define Partition Work Managers


Partition Work Managers ensure the fairness in thread resource usage and prioritization of work requests among partitions that share the same WLS instance.

You can define a Partition Work Manager in the domain to use with multiple domain partitions, or you can define Partition Work Manager attributes in the domain partition itself for use in that partition only. If no Partition Work Managers are defined, default values for Partition Work Manager settings are applied.

To define Partition Work Manager attributes in a domain partition for use in that partition only, see step 5 in Associate partitions with Partition Work Managers.

To define Partition Work Managers in the domain:

  1. If you have not already done so, in the Change Center of the Administration Console, click Lock & Edit (see Use the Change Center).
  2. In the left pane of the Console, expand Domain Partitions and select Partition Work Managers.

    The Partition Work Managers table summarizes the Partition Work Managers defined for domain partitions.

  3. Click New.
  4. On the Create a New Partition Work Manager page, provide a name for the Partition Work Manager and click OK.

    The new Partition Work Manager is created and displayed in the Partition Work Managers table.
  5. To activate these changes, in the Change Center of the Administration Console, click Activate Changes.
    Not all changes take effect immediately—some require a restart (see Use the Change Center).
